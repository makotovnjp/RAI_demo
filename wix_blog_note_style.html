1. ROS2の最新動向（2026年）

ROS2 Kilted Kaiju

2026年5月時点の最新LTS版は ROS2 Kilted Kaiju（サポート〜2026年11月）です。注目の更新点を挙げます。

  • Zenoh対応の安定化：DDS通信の代替として ROS2-Zenoh Bridge が正式サポート。低レイテンシ・省帯域の環境（エッジロボット、車載など）で恩恵が大きい
  • ros2_control の改善：ハードウェアインターフェースの抽象化が洗練され、産業用コントローラとの fieldbus 統合が容易に
  • Nav2 / MoveIt 2 の成熟：Dynamic obstacle avoidance が Nav2 に標準搭載。MoveIt 2 では Pilz Industrial Motion Planner が完全統合され、ROS1 時代のギャップがほぼ解消

ROS1 EOL 後の世界

PAL Robotics は 2026年4月1日をもって ROS1 スタックへの修正・新機能追加を終了し、すべての開発を ROS2 に移行すると発表しました。

「2019〜2021年には ROS2 を実用化するのが難しかったエコシステムのギャップが、2026年にはほぼ解消された」
— Robotics and Automation News (2026/04)

行動設計では Behavior Tree（BT）ベースのアーキテクチャが主流となり、LLM/VLMによるタスクプランニングの統合が次の主戦場になっています。ROSCon 2026 はカナダ・トロントで開催予定で、Physical AI と Embodied AI がホットトピックです。

2. RAIフレームワークとは

RAI は Robotec.ai が開発するオープンソースの Physical AI フレームワークです。「LLMにロボットの目と手を与える」ためのミドルウェア層と理解すると分かりやすいでしょう。

"RAI is a vendor agnostic agentic framework for Physical AI robotics, utilizing ROS 2 tools to perform complex actions, defined scenarios, free interface execution, log summaries, voice interaction and more."

2025年5月、arXiv に論文「RAI: Flexible Agent Framework for Embodied AI」（arXiv:2505.07532）として発表されました。

3層アーキテクチャ

  • Agents：run() / stop() を実装する基本単位。ReAct アーキテクチャの LLM として動作し、OpenAI・Anthropic Claude・ローカル LLM（Ollama）に対応
  • Connectors：センサーとアクチュエータの抽象化。ROS2 の topic / service / action を通じてロボットと通信する
  • Tools：LLM の tool-calling で呼び出せる関数群。@tool デコレータで独自ツールを追加可能

① rai_lite6_agent.py — メインエージェント

コードを GitHub で見る →

RAIのReActパターンを実装したメインファイルです。3つのクラスで構成されています。

Lite6Connector（ROS2通信層）：rclpy.node.Node を継承し、/ufactory/set_position サービスでデカルト移動、/ufactory/set_gripper_position でグリッパー制御を行います。_validate_pose() で作業空間の範囲外への移動を事前チェックする安全機構も実装しています。

# 使用例: ROS2サービスでLite6を制御
connector = Lite6Connector()
connector.initialize()  # Enable → Mode0 → State0
ok, msg = connector.move_to_pose(Pose(x=300, y=0, z=200), speed=100.0)
ポイント: RAI_SIM=1 環境変数を設定するとシミュレーションモードになり、実機なしで動作確認できます。

Lite6Tools（LLMツール定義層）：Anthropic の tool-calling 形式でロボット操作ツールを定義します。get_tool_definitions() が JSON スキーマを返し、execute_tool() がツール名と入力をディスパッチして実行します。

# LLMがこのツールを自律選択して呼び出す
tools = [
    {"name": "move_to_pose",    # デカルト座標への移動
     "name": "control_gripper", # グリッパー開閉
     "name": "get_camera_image",# カメラ画像取得（VLM解析用）
     "name": "get_joint_angles",# 関節角度読み取り
     "name": "move_home",       # ホームポジションへ帰還
     "name": "wait_seconds"}    # 待機
]

RAILite6Agent（LLMエージェント層）：Anthropic の messages.create() をReActループで繰り返し呼び出します。tool_use ブロックが返ってくる間はツールを実行して結果を会話履歴に追記し、end_turn で完了です。

# ReActループの核心部分
for _ in range(max_iterations):
    response = client.messages.create(
        model="claude-sonnet-4-6",
        system=SYSTEM_PROMPT,         # ロボット制御専用システムプロンプト
        tools=tools.get_tool_definitions(),
        messages=conversation_history,
    )

    if response.stop_reason == "end_turn":
        return response.content[0].text  # タスク完了

    # ツールを順番に実行して結果を会話に追加（→ 次のLLM判断へ）
    for block in response.content:
        if block.type == "tool_use":
            result = tools.execute_tool(block.name, block.input)
            conversation_history.append({"role": "user",
                "content": [{"type": "tool_result", "tool_use_id": block.id,
                              "content": result}]})

② pick_and_place_demo.py — VLM統合デモ

コードを GitHub で見る →

カメラ画像を Claude claude-sonnet-4-6（VLM）に送り、写っている物体を JSON 形式で返させてから自律的にピック＆プレースを行うデモです。

def detect_objects_with_vlm(client, image):
    """
    VLMで画像から把持可能な物体を検出する
    Returns: [{"name": "物体名", "image_position": "中央"}]
    """
    img_b64 = base64.standard_b64encode(
        cv2.imencode(".jpg", image)[1]
    ).decode("utf-8")

    response = client.messages.create(
        model="claude-sonnet-4-6",
        messages=[{"role": "user", "content": [
            {"type": "image", "source": {"type": "base64",
             "media_type": "image/jpeg", "data": img_b64}},
            {"type": "text",
             "text": '物体をJSON形式で列挙: {"objects": [...]}'},
        ]}],
    )
    return json.loads(response.content[0].text)
ポイント: 画像内の「左上・中央・右下」などのラベルを、あらかじめ定義したロボット座標テーブル（IMAGE_POS_TO_ROBOT）でmm単位に変換してツールに渡します。本格的な実装にはカメラキャリブレーション＋デプス情報が必要です。

# 簡易逆運動学でTCP位置からアーム各関節位置を計算
def compute_arm_segments(tcp_x, tcp_y, tcp_z):
    waist = np.arctan2(tcp_y, tcp_x)  # ウェスト回転角
    r = np.sqrt(tcp_x**2 + tcp_y**2) # 水平距離

    # 2リンクIK (上腕200mm + 前腕314mm)
    d = np.sqrt((r)**2 + (tcp_z - 243)**2)

        (d**2 - 200**2 - 314**2) / (2 * 200 * 314), -1, 1))

まとめ

ROS2 が産業用ロボット開発のデファクトスタンダードとなった 2026年において、RAI フレームワークは「LLM を使って ROS2 ロボットを自然言語制御する」ための現実的な選択肢です。

Connectors と Tools の抽象化設計により、ハードウェアが変わっても LLM 側のコードをほとんど変えずに対応できる点が大きなメリットです。UFACTORY Lite 6 から大型マニピュレータまで同じパターンで統合できます。
